## 60 秒版

> 面试官您好，我叫陈彬蔚，硕士毕业于香港科技大学化学与能源工程，本科毕业于浙江大学环境工程。我的背景一开始偏工程和科研，但读研和实习阶段逐渐转向 AI、深度学习和大模型应用。
>
> 最近我在香港科学园 Glassbox AI 参与 HSBC 香港手语无障碍项目，主要做中文和粤语文本到 HKSL Gloss JSON 的转换。这个项目不是简单调用 LLM，因为香港手语有自己的语法，金融场景又有专业术语，所以我们用了规则、术语库、检索、Prompt 约束、结构校验和专家反馈来形成一个可迭代的流程。
>
> 我在里面主要参与 Prompt、规则和术语知识库整理、不同 LLM 和 Embedding 模型效果对比，以及根据专家反馈调整检索和生成策略。这个经历让我比较直观地看到，大模型效果不只是模型本身，输入怎么组织、冗余信息怎么压缩、输出怎么约束、最后怎么评估，都会直接影响结果。
>
> 科研方面，我做过电池长期性能预测，用 FIR-LSTM 和 Transformer 处理电池循环数据。这里也有类似思路：不是直接把原始 time-series 电压曲线丢给模型，而是先重构成更适合预测的 n-series 表示，再做滤波、模型对比和外推效果评估。
>
> 所以如果总结我的经历，我比较关注的是：怎么把原始数据变成模型更容易利用的表示，怎么减少无效信息，怎么通过实验和反馈判断模型有没有变好。这个和 LLM 应用、Agent workflow、模型评估、文本压缩和推理优化这些方向是能接上的。

## 30 秒版

## 使用提醒

```text
1. 不要说自己是底层大模型训练专家。
2. 不要说自己完整做过 RL / 量化 / 剪枝 / 机器人控制。
3. 主线是：工程科研背景 -> AI/深度学习项目 -> HSBC LLM/RAG 项目 -> 电池序列建模 -> 数据表示、反馈评估、系统迭代。
4. 如果对方岗位偏 Agent / 自主智能，就强调 workflow、反馈闭环、状态管理和评估。
5. 如果对方岗位偏 LLM 算法，就强调 Transformer/序列建模、模型对比、文本压缩、评估和 bad case。
```
