# 面试追问总池

这个文件用来收纳聊天里出现过的追问。它不是标准答案文件，而是复习路线图：哪些问题已经沉淀，哪些问题还要继续开口练。

## 使用方式

```text
已沉淀：说明已经写入项目卡、深挖问答、概念补基础或模拟面试记录。
待继续练：说明有初步口径，但还需要继续模拟面试。
谨慎口径：说明面试时不能说太满，避免过度包装。
```

## 待继续练的问题

下面这些问题要继续用模拟面试的方式练。优先级从上到下。

### 1. Plan-and-Execute 是什么？

核心要回答：

```text
先整体规划，再逐步执行。
和 ReAct 的区别是：ReAct 边想边做；Plan-and-Execute 先制定计划，再按计划执行。
适合复杂、多步骤、需要全局规划的任务。
缺点是计划错了会影响后续，所以要允许重规划。
```

这个问题已经有基础答案，下一轮要练的是把它和 HSBC 项目区分开：你的项目更像 Agentic Workflow，不要硬说成完整 Plan-and-Execute Agent。

### 2. 自主智能系统核心模块怎么展开？

核心要回答：

```text
模型、感知、状态、规划、执行、记忆、评估、安全。
重点不是堆模块，而是讲清闭环：系统如何从目标出发，根据反馈调整行动，并用评估和安全边界保证可靠。
```

下一轮要练的是把这道题和你的 HSBC 项目接上：HSBC 有检索、fallback、校验、专家反馈，但不是完整自主智能，不要说太满。

## 谨慎口径

下面这些点面试时不要说太满。

```text
不要说“知识库是 LLM 生成的事实来源”。
更稳：LLM 辅助整理，事实来源是专家规则、术语表和反馈。

不要说“查不到就让模型自己编 Gloss”。
更稳：模型可以给候选草稿，正式 Gloss 必须经过术语库或专家确认。

不要主动说“用了 Milvus / pgvector”。
更稳：项目阶段主要用 Dify 内置知识库，后续可迁移。

不要把 20%+ 说成绝对精确值。
更稳：来自同一批 case 优化前后的专家可用性评估，具体数值按项目记录口径说。

不要把自己说成项目总架构师。
更稳：你参与 Prompt、知识库整理、Workflow 配置、测试和专家反馈闭环。
```
